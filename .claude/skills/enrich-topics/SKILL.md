# トピック充実化スキル

`topics.json` 内の特定カテゴリのトピックを分析し、網羅性・多様性・読者吸引力を高める
新トピックを生成して追加する。既存トピックとの重複を避け、カテゴリのブランクを埋める。

## 使い方

```
/enrich-topics [カテゴリ名] [--count N] [--dry-run]
```

**引数:**
- `[カテゴリ名]`: 対象カテゴリ（省略時は最もトピック数が少ないカテゴリを自動選択）
  - `道具・機器` / `equipment`
  - `抽出・レシピ` / `brewing`
  - `豆・産地` / `beans`
  - `コーヒー文化` / `culture`
  - `カフェ情報` / `cafe`
  - `ライフスタイル` / `lifestyle`
  - `焙煎・鮮度` / `roasting`
  - `健康・科学` / `science`
  - `世界の珍しいコーヒー` / `world`
  - `著名人とコーヒー` / `celebrity`
  - `コーヒー×ゲーム` / `game`
  - `時事・トレンド` / `trend`
- `--count N`: 追加するトピック数（デフォルト: 10）
- `--dry-run`: topics.json を更新せず、候補のみ表示

---

## 実行手順

### Step 1: 現状分析

`topics.json` を読み込み、以下を確認する:

1. 全カテゴリのトピック数を集計する
2. 対象カテゴリを決定する（引数省略時は最小件数のカテゴリ）
3. 対象カテゴリの既存トピックを列挙する

出力例:
```
📊 topics.json 現状分析
════════════════════════════════════
豆・産地         : 16件
抽出・レシピ     : 14件
道具・機器       :  9件  ← 対象カテゴリ
コーヒー文化     :  9件
...
════════════════════════════════════
対象カテゴリ: 道具・機器（9件）

既存トピック:
  - コーヒーミルの選び方
  - ハンドドリッパー比較 V60 vs Kalita
  ...
```

### Step 2: ブランク分析

既存トピックを以下の観点で分析し、カバーされていない領域を特定する:

**道具・機器カテゴリのカバレッジ観点:**
- グラインダー（手挽き・電動・業務用）
- ドリッパー（形状・素材・ブランド）
- エスプレッソマシン（家庭用・半業務用）
- ケトル（温度管理・容量・素材）
- スケール・計量器具
- 保存容器・キャニスター
- フィルター（ペーパー・メタル・布）
- サーバー・ドリップスタンド
- エスプレッソ補助器具（タンパー・WDT・ディストリビューター）
- メンテナンス・クリーニング用品
- 携帯器具・アウトドア向け
- ミルクフォーマー・スチームピッチャー
- 最新デジタル器具（スマートケトル・アプリ連動器具）

**抽出・レシピカテゴリのカバレッジ観点:**
- 基本抽出（ハンドドリップ・エスプレッソ・フレンチプレス）
- 器具別レシピ
- アレンジドリンク（ラテ・カプチーノ・マキアート等）
- スペシャルティ向けレシピ（バリスタ競技レシピ）
- 比率・温度・挽き目の科学的ガイド
- 季節別レシピ（アイス・ホット）

**豆・産地カテゴリのカバレッジ観点:**
- アフリカ（エチオピア・ケニア・タンザニア等）
- 中南米（コロンビア・ブラジル・グアテマラ・コスタリカ等）
- アジア（インドネシア・ベトナム・タイ等）
- 島嶼（ハワイ・ジャマイカ・パプアニューギニア等）
- スペシャルティ品種（ゲイシャ・ティピカ・ブルボン等）

### Step 3: 新トピック生成

ブランク分析に基づき、以下の品質基準で新トピックを生成する:

**品質基準:**
1. **具体性**: 抽象的でなく、読者が何を得られるか明確
2. **SEO友好性**: 検索意図を反映したキーワードを含む
3. **わくわくどきどき**: タイトルで好奇心を刺激する
4. **非重複**: 既存トピックと実質的に重複しない
5. **バランス**: カテゴリ内でサブテーマが偏らない

**タイトルパターン例（道具・機器）:**
- `<製品名> 完全レビュー <独自切り口>`
- `<器具A> vs <器具B> <比較軸>`
- `<課題> を解決する <器具カテゴリ> 選び方`
- `<マニアック器具> とは何か <分かりやすい説明>`
- `<価格帯>で揃える <器具セット> <ユースケース>`

### Step 4: 候補の表示と確認

生成した候補を表示し、ユーザーに確認を求める:

```
✨ 新トピック候補（道具・機器カテゴリ）
════════════════════════════════════
 1. <トピック名>
 2. <トピック名>
...
10. <トピック名>
════════════════════════════════════
全件 topics.json に追加しますか？
個別に選択する場合は番号をカンマ区切りで入力してください。
```

`AskUserQuestion` ツールを使って確認する。

`--dry-run` の場合はここで終了する。

### Step 5: topics.json の更新

ユーザーが承認したトピックを `topics.json` の該当カテゴリ末尾に追加する。

JSON形式:
```json
{ "topic": "<トピック名>", "category": "<カテゴリ名>" }
```

### Step 6: 完了レポート

```
✅ トピック充実化 完了
────────────────────────────────────
カテゴリ       : 道具・機器
追加トピック数  : <n>件
更新後件数     : <n>件（旧: <n>件）
────────────────────────────────────
💡 次のステップ
□ /article-cycle next で新トピックから記事生成
□ /enrich-topics <別カテゴリ> で他カテゴリも充実
```

## 設計方針

### 重複防止
既存トピックとコサイン類似度が高い（タイトルの主語・比較対象・テーマが同一）候補は自動除外する。

### カテゴリ横断整合性
同じ器具・製品が別カテゴリに既に存在する場合（例: 抽出・レシピカテゴリに「エアロプレス」）は、
そのカテゴリでは「器具レビュー」視点、別カテゴリでは「使い方・レシピ」視点として差別化する。

### トピック粒度
一記事で書ける範囲を意識する。広すぎるテーマ（「エスプレッソマシン完全ガイド」）は
複数の具体的トピックに分割する。
