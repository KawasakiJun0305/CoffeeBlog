# 記事自動生成サイクル管理スキル

コーヒーブログ記事を体系的・サイクリックに継続生成するスキル。
`coffee-knowledge` で蓄積した知識ベースからバックログを構築し、
ドメインバランスと季節性を考慮しながら週2〜4本の記事を自動選択・生成・記録する。

**ターゲット**: 神奈川在住の読者が毎日わくわくどきどきできる記事を届けるため、
「世界のコーヒー探訪」「日本最新コーヒーシーン」「神奈川コーヒーマップ」の3シリーズを
ローテーションに組み込む。

## 使い方

```
/article-cycle [コマンド] [オプション]
```

**コマンド（省略時は `next`）:**
| コマンド | 説明 |
|---------|------|
| `init` | バックログを初期化（knowledge/ から候補を収集） |
| `next` | 次の記事を自動選択して生成する |
| `plan` | 今週〜次週の投稿計画を表示 |
| `report` | サイクル進捗・ドメインカバレッジを表示 |
| `add <トピック>` | バックログに手動でトピックを追加 |

**オプション:**
| オプション | 説明 |
|-----------|------|
| `--week N` | N週間分の計画を表示（デフォルト: 2） |
| `--review` | 生成後に品質スコアリングを実施（generate-coffee-article の --review を付与） |
| `--dry-run` | 記事を生成せず、選択結果・計画のみ表示 |
| `--force <トピック>` | アルゴリズムを無視して指定トピックを強制生成 |

---

## 実行手順

---

### `init` — バックログ初期化

#### Step 1: 既存リソースの確認

以下のファイルが存在するか確認する:

```
knowledge/
  INDEX.md           ← coffee-knowledge が生成するインデックス
  beans.md           ← 各ドメインの知識 + 記事アイデア
  processing.md
  roasting.md
  brewing.md
  sensory.md
  equipment.md
  culture.md
  science.md
  topic-backlog.md   ← このスキルが管理するバックログ（新規作成）
  content-calendar.md ← このスキルが管理するカレンダー（新規作成）
```

`knowledge/` の `*.md` が存在しない場合:
```
⚠️  knowledge/ ディレクトリが空です。
先に /coffee-knowledge を実行して知識ベースを構築してください。
```
と表示して終了する。

#### Step 2: 各ドメインファイルから記事候補を収集

各 `knowledge/<domain>.md` の「記事アイデア」セクションを読み込み、
全トピックを収集する。

ドメインとカテゴリのマッピング:
| ドメイン | generate-coffee-article カテゴリ |
|---------|-------------------------------|
| world-origins | 世界のコーヒー探訪 |
| japan-scene | 日本最新コーヒーシーン |
| kanagawa | 神奈川コーヒーマップ |
| beans / processing | 豆・産地 |
| roasting | 豆・産地 |
| brewing | 抽出・レシピ |
| sensory | コーヒー文化 |
| equipment | 道具・機器 |
| culture | コーヒー文化 |
| science | コーヒー文化 |

#### Step 3: topic-backlog.md を生成

`knowledge/topic-backlog.md` を以下の形式で作成する:

```markdown
# コーヒーブログ 記事バックログ

> 最終更新: <日付> | 管理: article-cycle スキル

## 優先度凡例
⭐⭐⭐ 高（季節性あり・検索需要大） / ⭐⭐ 中 / ⭐ 低

## ステータス凡例
- `[ ]` 未生成（pending）
- `[>]` 生成済み
- `[S]` スキップ（季節外・重複）
- `[📅]` 計画済み（生成予定日あり）

---

## 🌱 豆・産地カテゴリ
| ステータス | 優先度 | トピック | 補足 |
|-----------|-------|---------|------|
| [ ] | ⭐⭐⭐ | エチオピア イルガチェフェ | 定番・人気高 |
| [ ] | ⭐⭐ | ... | ... |

## ☕ 抽出・レシピカテゴリ
...

## 🏠 コーヒー文化カテゴリ
...

## 🛠 道具・機器カテゴリ
...

## カフェ情報カテゴリ
...

---

## 生成済み記事ログ
| 日付 | トピック | カテゴリ | ファイル名 | 品質スコア |
|------|---------|---------|----------|-----------|
（生成するたびに追記）
```

#### Step 4: content-calendar.md を初期化

```markdown
# コンテンツカレンダー

> 管理: article-cycle スキル | 投稿目標: 週2〜4本

## 今週の計画
（article-cycle plan で自動生成）

## 先週の実績
（記録なし）
```

完了メッセージ:
```
✅ バックログ初期化完了
────────────────────────────────────
収集トピック数 : <n>件
カテゴリ内訳   :
  豆・産地      : <n>件
  抽出・レシピ  : <n>件
  コーヒー文化  : <n>件
  道具・機器    : <n>件
  カフェ情報    : <n>件
────────────────────────────────────
次のステップ: /article-cycle next で記事生成を開始
```

---

### `next` — 次の記事を自動選択・生成

#### Step 1: バックログを読み込む

`knowledge/topic-backlog.md` を読み込む。
ファイルが存在しない場合は `init` を先に実行するよう案内する。

#### Step 2: ドメインローテーション判定

「生成済み記事ログ」の直近4件のカテゴリを確認する。

**ローテーションルール（月別目標比率）:**
| カテゴリ | 月の目標比率 | 週の目安 |
|---------|------------|---------|
| 世界のコーヒー探訪 | 25% | 月2〜3本 |
| 日本最新コーヒーシーン | 20% | 月2本 |
| 神奈川コーヒーマップ | 15% | 月1〜2本 |
| 豆・産地 | 15% | 月1〜2本 |
| 抽出・レシピ | 15% | 月1〜2本 |
| 道具・機器 / コーヒー文化 / カフェ情報 | 10% | 月1本 |

- 直近4件のうち同カテゴリが2件以上 → そのカテゴリを今回はスキップ
- 「世界のコーヒー探訪」は産地カバレッジリストの未カバー産地を優先
- 「日本最新コーヒーシーン」は常に最新情報が必要 → `--japan-trend` フラグ必須
- 「神奈川コーヒーマップ」は季節に合ったエリアを優先

例:
```
直近4件: 豆・産地, 抽出・レシピ, 世界のコーヒー探訪, 日本最新コーヒーシーン
→ 神奈川コーヒーマップ が最も少ない → 次は神奈川を優先
```

#### Step 3: 季節性フィルタリング

現在月に基づいて季節性トピックを優先する。神奈川読者の行動パターンを意識する。

| 月 | 優先トピック傾向 | 神奈川ローカル角度 |
|----|----------------|-----------------|
| 12〜2月 | ホットコーヒー・深煎り・冬カフェ・ギフト | 箱根の温泉カフェ・横浜イルミコーヒー |
| 3〜5月 | 新生活・初心者向け・春の新豆 | 鎌倉の桜×コーヒー・湘南テラス席 |
| 6〜8月 | アイスコーヒー・水出し・コールドブリュー | 江ノ島海辺カフェ・茅ヶ崎ビーチコーヒー |
| 9〜11月 | 新豆・ハーベスト・秋の深煎り | 箱根紅葉カフェ・鎌倉古民家カフェ |

季節タグが付いたトピックは優先度を1段階引き上げる。

#### Step 4: 選択結果を表示

```
📋 次の記事候補
────────────────────────────────────
選択トピック : <トピック名>
カテゴリ     : <カテゴリ>
優先度       : ⭐⭐⭐
選択理由     : <ドメインバランス調整 / 季節性考慮>
────────────────────────────────────
```

`--dry-run` の場合はここで終了する。

#### Step 5: 記事生成

`generate-coffee-article` スキルを呼び出して記事を生成する。

呼び出しパラメータ:
```
generate-coffee-article <トピック> <カテゴリ> [--review（オプション指定時）]
```

#### Step 5.5: リーガルチェック（必須・自動実行）

`generate-coffee-article` の生成完了直後に `legal-check-coffee-article` スキルを自動実行する。
**ユーザーが `--skip-legal` を明示的に指定した場合を除き、スキップ禁止。**

```
legal-check-coffee-article <output/生成ファイルパス>
```

スコア別対応:

| スコア | 対応 |
|--------|------|
| 75〜100 | Step 6（バックログ更新）へ進む |
| 60〜74  | 🔴項目を修正し **リーガルチェックを再実行** してから Step 6 へ |
| 0〜59   | バックログを `[>]` に更新せず `[⚠️法的修正待ち]` として記録する |

チェック結果をバックログ更新時の「品質スコア」列に記録する:
```
| <日付> | <トピック> | <カテゴリ> | <ファイル名> | 品質: __/25 / 法的: __/100 |
```

---

#### Step 6: バックログを更新

生成完了後、`knowledge/topic-backlog.md` を更新する:
- 該当トピックのステータスを `[ ]` → `[>]` に変更
- 生成済み記事ログに追記:

```
| <今日の日付> | <トピック> | <カテゴリ> | <output/ファイル名> | <品質スコア or —> |
```

#### Step 7: カレンダー更新

`knowledge/content-calendar.md` の「今週の計画」または「実績」に追記する。

---

### `plan` — 投稿計画の表示・生成

#### Step 1: バックログと現在のカレンダーを読み込む

#### Step 2: N週間分のスケジュールを生成

週ごとに「週2〜4本」の目標に従い、ドメインローテーションと季節性を考慮して
バックログから候補を自動選択して計画を表示する。

```
📅 投稿計画（今週〜<N週後>）
────────────────────────────────────
【第1週】2026/06/16〜06/22（目標: 3本）
  月 06/16  豆・産地    エチオピア イルガチェフェの魅力
  水 06/18  抽出・レシピ  ハンドドリップ入門〜初心者でも失敗しない3ステップ
  金 06/20  道具・機器   Commandante C40 レビュー

【第2週】2026/06/23〜06/29（目標: 3本）
  月 06/23  コーヒー文化  サードウェーブとは何だったのか
  水 06/25  抽出・レシピ  水出しコーヒーの作り方
  金 06/27  豆・産地    コロンビア ウィラ産の特徴
────────────────────────────────────
計画をカレンダーに保存しますか？ [y/n]
```

ユーザーが `y` を選んだ場合は `content-calendar.md` に書き込む。

### `add <トピック>` — 手動でトピックを追加

ユーザーが指定したトピックをバックログに追加する。

カテゴリが不明な場合は `generate-coffee-article` のカテゴリ推定ルールに従って自動推定し、
ユーザーに確認を求める（`AskUserQuestion` ツールを使うこと）。

追加後メッセージ:
```
✅ バックログに追加しました
  トピック : <トピック>
  カテゴリ : <カテゴリ>
  優先度   : ⭐⭐（デフォルト）
```

## サイクル設計原則

### ドメインバランス目標

全生成記事のカテゴリ比率は以下を目標とする（神奈川×世界×日本最新版）:
```
世界のコーヒー探訪    : 25%  ← 産地カバレッジを毎月更新
日本最新コーヒーシーン : 20%  ← 鮮度が命、--japan-trend 必須
神奈川コーヒーマップ   : 15%  ← 月1〜2本、季節ローテーション
豆・産地              : 15%
抽出・レシピ          : 15%
道具・機器 + コーヒー文化 + カフェ情報 : 10%
```

比率が大きく乖離している場合は `report` で警告し、`plan` の選択アルゴリズムで自動補正する。

### 世界産地カバレッジ管理

`knowledge/world-origins.md` の産地カバレッジリストを参照し、
未カバーの産地を優先的にバックログに追加する。
年間で全主要産地（20産地以上）を一巡することを目標とする。

### 重複防止ルール

同じトピックを再生成しないよう、生成済みログとバックログのステータスを常に参照する。
類似トピック（例: 「ハンドドリップ 基本」と「ハンドドリップ 入門」）も人間が確認できるよう
バックログに備考として記載する。

### knowledge/ との連携

`coffee-knowledge` スキルでドメインが更新されたら、
そのドメインの「記事アイデア」を読み込んでバックログに差分追加する。
（`init --merge` オプション: 既存バックログを保持しつつ新トピックのみ追加）

## 依存スキル

| スキル | 用途 | 必須 |
|--------|------|------|
| `coffee-knowledge` | 知識収集・記事候補の生成元（world-origins / japan-scene / kanagawa ドメイン含む） | 任意 |
| `generate-coffee-article` | 個別記事の生成（--japan-trend / --kanagawa フラグ対応） | 必須 |
| `legal-check-coffee-article` | 法的リスクチェック（景品表示法・薬機法・著作権法等）**next 実行時に自動呼び出し** | **必須** |
| `web-search` | 季節トレンドの確認・日本最新情報収集（plan 実行時） | 任意 |

| `knowledge/content-calendar.md` | 週別投稿計画と実績 |
