# コーヒー記事 高品質生成スキル

CoffeeBlog（note.com向けコーヒー専門ブログ）の高品質記事を生成するプロジェクト専用スキル。
神奈川在住の読者が**毎日わくわくどきどき**できる記事——世界のコーヒー産地への旅感、
日本最新シーン、神奈川ローカル情報——を生成する。

## 使い方

```
/generate-coffee-article <トピック> [カテゴリ] [--research] [--review] [--japan-trend] [--kanagawa]
```

**引数:**
- `<トピック>`: 記事テーマ（例: "エチオピア イルガチェフェ", "横浜元町のカフェ散策"）
- `[カテゴリ]`: 以下から選択（省略時はトピックから自動推定）
  - `世界のコーヒー探訪` / `日本最新コーヒーシーン` / `神奈川コーヒーマップ`
  - `豆・産地` / `抽出・レシピ` / `カフェ情報` / `道具・機器` / `コーヒー文化`
- `--research`: Web検索で最新情報を収集してから生成
- `--review`: 生成後に品質スコアリングと改善を実施
- `--japan-trend`: 日本の最新コーヒートレンドをWeb検索で収集して記事に反映
- `--kanagawa`: 神奈川ローカル情報（カフェ・アクセス・エリア感）を優先的に盛り込む

---

## 実行手順

### Step 0: 引数解析とカテゴリ推定

ユーザーのメッセージからトピック・カテゴリ・オプションフラグを読み取る。
カテゴリ省略時はトピックの内容から以下のルールで自動推定する:

- 産地名・国名・農家・コーヒー産地の旅 → `世界のコーヒー探訪`
- 新規オープン・受賞・チャンピオン・トレンド・イベント → `日本最新コーヒーシーン`
- 神奈川・横浜・鎌倉・湘南・川崎・箱根・小田原 → `神奈川コーヒーマップ`
- 豆名・品種・焙煎度・精製方法 → `豆・産地`
- 抽出方法・レシピ・淹れ方 → `抽出・レシピ`
- カフェ名・店名・〜巡り（地域特定なし） → `カフェ情報`
- 器具名・グッズ・マシン → `道具・機器`
- 歴史・文化・〜とは（トレンド除く） → `コーヒー文化`

---

### Step 1: 情報収集（フラグ指定時）

#### `--research` 指定時
`web-search` スキルで以下を調査してからStep 2へ:
1. トピックに関する最新情報・正確なファクト（産地情報・数値・固有名詞）
2. note.comで人気のコーヒー記事タイトルのパターン
3. 競合記事にない独自の切り口

#### `--japan-trend` 指定時
`web-search` スキルで以下を調査:
1. 直近のJBC/WBC（バリスタチャンピオンシップ）結果・話題
2. 日本の新規オープン・注目スペシャルティカフェ情報
3. 日本のコーヒー関連ニュース（2026年）

#### `--kanagawa` 指定時
`web-search` スキルで以下を調査:
1. 神奈川の話題カフェ・ロースター情報
2. 横浜・鎌倉・湘南・箱根エリアのカフェ開店・閉店情報
3. 神奈川発コーヒーイベント情報

---

### Step 2: カテゴリ別強化プロンプトで記事生成

`prompts/coffee-article.md` の対応カテゴリのプロンプトを使って記事を生成する。
以下の**共通品質基準**を全カテゴリで満たすこと。

---

#### 共通品質基準（全カテゴリ共通）

- **タイトル**: 30文字以内。数字・疑問形・「実は」「〇〇の理由」が効果的
- **リード文（冒頭3〜4文）**: 「驚き・旅心・共感・行動欲求」のどれかで読者を引き込む
- **発見の瞬間**: 「え、知らなかった！」という気づきを1箇所以上入れる
- **行動喚起**: 「今週末行ってみよう」「今日試してみよう」につながる締めを1文で入れる。二重締め禁止
- **具体性**: 数値・固有名詞・地名・エピソードを必ず含める（「ある程度」はNG）
- **note.comプレビュー**: 冒頭140文字が単独で意味をなし、読者の興味を引く
- **文体**: です・ます調、親しみやすさ×専門知識のバランス

---

#### 世界のコーヒー探訪（1800〜2200字）

```
あなたは世界のコーヒー産地を旅してきたスペシャルティコーヒーの専門家兼ライターです。
神奈川在住の読者に向けて、コーヒーを通じて世界を旅する感覚を届けてください。

{{トピック（産地・農家・旅エピソード）}}について、
読者が「このコーヒーを飲んで旅した気分になれる」旅行記スタイルで書いてください。

【構成】
1. 旅の始まり（その産地の情景や出会いのエピソードから始める）
2. 産地の素顔（地理・標高・気候・人々の暮らし——数値と情景を交えて）
3. 農家・生産者のストーリー（人の顔が見える記述）
4. この土地にしか出せない風味（具体的な食べ物・飲み物で表現）
5. 神奈川・日本でこの産地のコーヒーに出会える場所（ヒント提示）
6. 締め（「次の一杯で、〇〇に旅してみてください」）

【禁止】
- 「〜かもしれません」「〜のようです」などの曖昧な逃げ表現
- 根拠のない断定
```

#### 日本最新コーヒーシーン（1400〜1800字）

```
あなたは日本のコーヒーシーンを追いかける専門ライターです。
神奈川在住の読者に、今の日本で起きていることをリアルタイム感たっぷりに届けてください。

{{トピック（新店・受賞・トレンド・イベント等）}}について書いてください。

【構成】
1. ニュース的な導入（「今、日本のコーヒー界で注目されているのが〜」）
2. 背景・文脈の解説（なぜ今これが話題なのか）
3. 具体的な情報・事例（店名・人名・数値など実例2〜3個）
4. 神奈川・首都圏の読者がアクセスできるか（地理的な視点）
5. これからどうなるか（展望）
6. 締め（「あなたのコーヒーライフにも取り入れてみませんか」）

【必須】
- 「今」「2026年」「最新」などの時制を意識した表現
- 実在する店名・人名・ブランド名（架空はNG）
```

#### 神奈川コーヒーマップ（1200〜1600字）

```
あなたは神奈川在住のコーヒー専門ライターで、地元のカフェを知り尽くしています。
神奈川在住の読者が「今週末実際に行ける」情報として書いてください。

{{トピック（エリア・カフェ名・テーマ）}}について書いてください。

【構成】
1. そのエリア・テーマを一言で表すキャッチコピー
2. なぜ今このエリア・カフェが面白いのかの背景
3. 具体的なスポット・ルート情報（2〜3ヶ所）
4. 各スポットの雰囲気・おすすめの一杯・見どころ
5. アクセスと行き方のヒント（最寄り駅・所要時間の目安）
6. 締め（「ぜひ今週末、出かけてみてください」）

【注意】
- 価格・営業時間は「公式SNS・サイトでご確認ください」と必ず添える
- 実在確認できるスポットのみ記載
```

#### 豆・産地 深掘り（1800〜2000字）

```
あなたは10年以上の経験を持つスペシャルティコーヒーの専門家兼ライターです。
神奈川在住のコーヒー初心者〜中級者向けに、「このコーヒーを選ぶ理由が分かった！」
という知的満足感を与える記事を書いてください。

{{トピック（豆名・品種・精製方法等）}}について、1800〜2000字で書いてください。

【構成】
1. フック（最も面白いポイントを一文で）
2. 産地の背景と歴史（標高・気候・農家のストーリーを交えて）
3. 品種・精製方法の解説（専門用語は必ず平易な説明を添える）
4. 風味プロファイル（具体的な食べ物・飲み物で表現）
5. おすすめの焙煎度と抽出方法（具体的な数値: 温度・量・時間）
6. 神奈川・日本でどこで手に入れられるか（ヒント）
7. 締め（「一度飲んだら、コーヒーへの見方が変わります」）
```

#### 抽出・レシピ（1600〜2000字）

```
あなたはバリスタの技術を持つコーヒーライターです。
神奈川の自宅でも今日から再現できるレシピ記事を書いてください。

{{トピック（抽出方法・レシピ名等）}}について、1600〜2000字で書いてください。

【構成】
1. フック（この方法で何が劇的に変わるかを1文で）
2. 難易度と所要時間（冒頭に明記: 難易度★★☆☆☆ / 準備〇分+抽出〇分）
3. 必要な器具・材料リスト（代替品も提示）
4. ステップバイステップのレシピ（温度・量・時間の数値必須）
5. 失敗しやすいポイントとQ&A（2〜3つ）
6. バリエーション・アレンジ提案
7. 締め（「今日帰ったら、ぜひ試してみてください」）
```

#### カフェ情報（1200〜1500字）

```
あなたは国内外のカフェを訪れてきたコーヒーブロガーです。
読者が今すぐ行きたくなるレビュー記事を1200〜1500字で書いてください。

{{トピック（カフェ名・エリア等）}}について書いてください。

【構成】
1. そのカフェを一言で表すキャッチコピー
2. 雰囲気・内装の描写（写真なしでも目に浮かぶ具体的な描写）
3. おすすめドリンク・フードの詳細（味・見た目・価格帯）
4. おすすめシーン（一人作業・デート・週末散歩等）
5. アクセスと注意事項（価格・営業時間は「公式サイトでご確認ください」）
6. 総合評価（★5つで採点 + 一言コメント）
```

#### 道具・機器（1600〜2000字）

```
あなたはコーヒー器具マニアのレビュアーです。
読者の購入判断に役立つ詳細レビュー記事を1600〜2000字で書いてください。

{{トピック（器具名等）}}について書いてください。

【構成】
1. この器具が解決する「悩み」（読者の共感を呼ぶ導入）
2. スペック・外観の詳細（サイズ・素材・価格帯）
3. 実際の使い心地（良い点3つ・惜しい点2つ）
4. 他製品との比較（2〜3製品を簡易比較表で）
5. こんな人に向いている / 向いていない（チェックリスト形式）
6. 総合評価と購入おすすめ度（★5つ + 最終コメント）

【必須】価格帯の目安を記載（正確な値段は「公式サイト参照」）
```

#### コーヒー文化（1800〜2000字）

```
あなたはコーヒーの文化・歴史に詳しいジャーナリストです。
読者の知的好奇心を刺激する読み物記事を1800〜2000字で書いてください。

{{トピック（文化・歴史等）}}について書いてください。

【構成】
1. 意外な事実や驚きの導入（「実は〇〇だった」系のフック）
2. 背景・歴史・現状の解説
3. 日本・神奈川の読者の日常との関連付け（「身近なあの〜も実は…」）
4. トレンドや将来の展望
5. 読者が今日からできること（具体的なアクション提案）
6. エモーショナルな締め（コーヒーの魅力を再発見させる一文）

【禁止】うんちくの羅列（常に「読者にとっての意味・価値」を示す）
```

### Step 3.5: リーガルチェック（必須・スキップ禁止）

`legal-check-coffee-article` スキルを呼び出して法的リスクをチェックする。
**これは全記事に対して必ず実施する。`--review` フラグの有無に関わらず実行する。**

スコア別対応:

| スコア | 対応 |
|--------|------|
| 90〜100 | 次のステップへ進む |
| 75〜89  | 🟡項目を修正してから次のステップへ |
| 60〜74  | 🔴項目を修正し **リーガルチェックを再実行** する |
| 0〜59   | 公開不可。記事を大幅修正または再生成する |

カテゴリ別重点チェック:
- カフェ情報 / 神奈川コーヒーマップ: Check 2（ステマ）・Check 4（誹謗中傷）・Check 5（店舗情報）
- 健康・栄養への言及がある記事: Check 1 のスコアが 18/20 未満なら即修正
- 道具・機器: Check 7（比較広告）・Check 2（ステマ）

---

### Step 4: 品質スコアリング（--review 指定時）

生成した記事を以下で評価し、スコアが18未満なら自動改善する:

```
品質スコアカード:
□ タイトルの魅力度     (1-5): _ / 5  ← クリックしたいか
□ リード文の引力       (1-5): _ / 5  ← 3文で引き込めているか
□ 構成の明確さ         (1-5): _ / 5  ← 見出しで流れがわかるか
□ 具体性・信頼性       (1-5): _ / 5  ← 数値・固有名詞・地名があるか
□ わくわくどきどき感   (1-5): _ / 5  ← 発見・行動喚起があるか
────────────────────────────────────
合計: _ / 25

20以上: 合格（保存へ）
17〜19: 軽微改善後に保存
16以下: タイトル・リード・締めを再生成
```

---

### Step 5: ファイル保存

以下のコマンドで output/ に保存する:

```bash
npm run generate -- "<トピック>" "<カテゴリ>"
```

または、生成した記事を `output/<日付>-<トピック>.md` に直接書き込む。
